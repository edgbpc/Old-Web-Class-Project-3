No changes to responsive design from project 1.
Required Fields still handed by HTML so the PHP code that triggers "Required Field" error will never execute. I would have preferred to let PHP handled that for this assignment but had issues that my form would submit even if some of the required fields weren't filled out. Using HTML solved this issue but did make the PHP code unncessary.
For project 3, I intend to let javascript and PHP handle the required fields so hopefully will be removing that attribute from the HTML code for that turn in. For now, I left the code in place
I had intended to keep as many of my PHP functions in a seperate script file but found it much easier to ensure variables were being assigned and cleared out correctly with that code remaining on the main registration PHP page
I do still use some Javascript but all validation is now handled by PHP.
I opted not use the Filter available in PHP for email validation because I wanted the same logic to be used in javascript as in PHP so that Javascript would OK an email address just to have PHP reject it
I used the Sessions array to pass variables into a confirmation page. I do still use some of the javascript from the first project. Auto format for phone and zipcode are still handed by the javascript. The turn in file for javascript has all the validation code commented out. Changed the drop down for states so there is always a state selected instead of "Select State" being the default selected option.
